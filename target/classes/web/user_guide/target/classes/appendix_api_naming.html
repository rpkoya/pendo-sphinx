Appendix - API Naming

Appendix - API Naming-conventions

A few rules help keep the system humming along. When (re-)naming API Endpoints, there are a few non-obvious rules that should be followed.

  • Please only use alphanumeric characters and the hyphen (-). Do not use other special characters or spaces.
  • Please use camelCase (a.k.a camel caps or medial capitals). This is the practice of writing compound words or phrases such that each word or abbreviation begins with a capital letter. In the PD3P, as is often the case in programming languages, start with a lowercase letter. For example bondYieldMethodology, bookingDate, loan1, loan2 and brokerNamr.
  • Plural Names: the system needs both a singular name and a plural name for Entities. In general, when the system wants to find something, it uses the singular, and if it wants to list something, it uses the plural. Pluralization is typically a systemic function (the system code already uses a pluralizer). However, you may wish to override the generated plural name in some cases.
